Debrief

01 Input is not data

The model sees one flat stream of text. A user message can carry instructions that override the system prompt — that's prompt injection. The root cause is the missing trust boundary between instructions and input.

02 Blocklists are brittle

Filtering keywords like ignore (tier L2) is trivially bypassed with paraphrase, role-play, or encoding. Defense needs layers, not a wordlist.

03 Don't trust the model with secrets

The durable fix isn't a smarter prompt — it's architecture. Keep secrets and privileged tools out of the model's reach: least privilege, output filtering, and human approval for sensitive actions.

Workshop note  Offline simulation — the flag is fabricated and no live model is involved, so ARGUS's behaviour is scripted and repeatable. Map each technique to its real counterpart: indirect injection via poisoned RAG documents or web pages, jailbreaks against chat assistants, and tool-calling agents tricked into unauthorized actions. Good discussion prompts: where is the trust boundary? what could ARGUS not leak if the flag simply weren't in its context?
